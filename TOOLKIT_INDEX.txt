FOR DEVELOPERS READING THIS
================================================================================

Architecture Notes:
  - calc-questions.js stores all written question data
  - Questions grouped by lecture ("lec1" through "lec7", skipping "lec5")
  - Each question has metadata + image paths + fields to calculate
  - Images stored in questions_data/chX/Y/ directories
  - Field "tolerance" accepts relative value ±X from answer

To extend this toolkit:
  1. Copy LECTURE7_MISSING_QUESTIONS_TEMPLATE.txt
  2. Modify for new lecture/examples
  3. Add new chapter to expected_questions in audit script
  4. Test audit script identifies correctly
  5. Document in README file

Common modifications:
  - Adding new lectures
  - Changing tolerance calculation
  - Supporting different image formats
  - Adding more metadata fields
  - Supporting multiple languages for labels

IMPORTANT NOTES FOR AI MODELS
================================================================================

When following these guides:

✓ SHOULD DO:
  - Read entire relevant document before starting
  - Verify each step before moving to next
  - Test in browser after making changes
  - Check JSON syntax with validator
  - Take screenshots at 150+ DPI
  - Document what you did

✗ SHOULD NOT DO:
  - Skip validation steps (causes hours of debugging)
  - Copy-paste without understanding
  - Assume directory structure is wrong
  - Guess at answer values without checking PDF
  - Mix up ID numbers (esp. ch7/10 vs ch7/1)
  - Use automated extraction without verification

Common Pitfalls:
  - Thinking ch7/10 should be ch7/1/0 (WRONG - use 10)
  - Forgetting to add trailing comma in JSON
  - Using wrong PDF file (Lec6 instead of Lec7)
  - Not checking if image files actually exist
  - Assuming PDF page numbers = example numbers

When Stuck:
  1. Re-read the relevant section carefully
  2. Check that all files actually exist
  3. Validate JSON with online tool first
  4. Check browser console (F12) for errors
  5. Run audit script to see current status
